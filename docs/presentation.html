Architecture

Two Agents, Two Triggers

monitor-agent

Proactive Daily Intelligence

Runs at 6am every day. No input needed. Scans all markets, detects issues, delivers Decision Packets before anyone asks.

Trigger  → EventBridge 6am cron
Input    → None required
Output   → Decision Packets · Alerts
analytics-agent

On-Demand Q&A

Ask anything in plain English. The agent decides which tools to call, in what order, how many times — and returns a structured answer a VP can act on.

Trigger  → Natural language question
Input    → Any market / deal / risk
Output   → Insight · Evidence · Action

Both agents run on Claude Opus 4.6 · Native tool use · No wrappers · Anthropic SDK only

Agent Reasoning

Four Steps, Every Response

1

FRAME

State competing hypotheses before touching any data. Pricing issue? Demand issue? Supply surge?

2

ANALYZE

Call tools in sequence. Loop until there is enough evidence. Rule hypotheses in or out.

3

VALIDATE

Check consistency, math, severity, and confidence before writing a single word of output.

4

OUTPUT

Decision Packet with owner, deadline, dollar impact, and confidence score attached.

# Example framing before any tool call
Frame:
  H1: Pricing drift    — offers above AVM, suppressing acceptance   → test: LSR + offer/market spread
  H2: Demand softening — buyer pool shrinking, independent of price   → test: lead volume + DOM
  H3: Supply surge     — inventory building faster than absorption   → test: months of supply

  Testing H1 first — most likely given current rate volatility.

Output Contract

What a Decision Packet Looks Like

DECISION PACKET ─────────────────────────
Issue:     Offer acceptance dropped 11.8% WoW
Severity:  HIGH — act within 72 hours

Evidence:
  · LSR at 0.94 (threshold: 0.95)
  · Offers drifting +4.2% above AVM
  · DOM increased 6 days MoM
  · Lead volume holding — problem at offer stage

Impact:
  · ~10 homes/month not closing
  · $192,000 margin at risk / month
  · Annualized exposure: ~$2.3M

Action:
  1. Narrow offer band -2% to -4% by Friday
  2. Segment by property type
  3. Monitor LSR weekly for recovery

Expected outcome: +6–9% acceptance in 2–3 weeks
Owner:   Pricing Team
Confidence: High (feedback-calibrated: 0.82)

Not an insight

Most tools surface signals. This delivers an action plan with a named owner, a deadline, and the dollar cost of inaction.

Dollar-grounded

Every signal gets translated into homes/month and margin dollars before it reaches a human. No abstract percentages.

Confidence-scored

Outputs are validated before delivery. Confidence is feedback-calibrated — it improves as outcomes are tracked.

Design Principle

The Semantic Layer — One Definition Per Metric

What it is

The tool layer acts as a governed metric contract for both humans and the AI. Claude never touches raw data — it always goes through a tool that owns the business logic.

Why it matters

AI makes natural-language analytics faster. It does not make semantic discipline optional. Without it, you don't get hallucination — you get metric drift. One prompt pulls one source. Another pulls another. The meeting turns into a debate about definitions.

# One definition, used everywhere
CM_FLOOR           = 30_000    # $30k min per deal
TARGET_HOLD_DAYS   = 90        # Capital-Light target
LSR_THRESHOLD      = 0.95      # acceptance trigger
HOLDING_COST_PCT   = 0.00025   # ~$230/day per property

# Claude never does this:
SELECT * FROM raw.offer_events  ← never

# Claude always does this:
analyze_pricing_accuracy(market="Phoenix")
→ governed calculation → trusted number

# Result: "$192k at risk" is a fact,
#         not a guess.

Self-Improving System

The Feedback Loop

Decision Packet generated
        ↓
log_recommendation()    ← prediction stored with confidence score
        ↓
  operator takes action  (or doesn't)
        ↓
record_outcome()         ← actual KPI measured 2–3 weeks later
        ↓
recalibrate_confidence() ← predicted vs actual compared
        ↓
next briefing is smarter ← weights adjusted per issue type

Tracks predictions

Every Decision Packet is logged with the expected outcome and confidence score before it goes out.

Measures reality

When the actual KPI is measured, prediction accuracy is computed automatically.

Gets smarter

Confidence weights recalibrate per issue type. No manual retraining — it learns from outcomes.
